# MetaPlogging MVP 기능 명세

## 개요

플로깅(plogging) 활동 중 수거한 쓰레기를 사진으로 기록하고 업로드하는 데 최적화된 앱.  
MVP의 핵심 가치는 **빠르고 간편한 사진 수집**과 **데이터 품질 보장**이다.

---

## 기능 1. 사진 업로드

### 목표
촬영 → 업로드까지의 단계를 최소화한다.

### 동작 방식

**경로 A — 카메라 직접 촬영**
1. 메인 화면의 촬영 버튼 탭
2. 카메라 앱 실행 → 촬영
3. 이미지 정제 검사 실행 (중복·흔들림 판별)
4. 검사 통과 시 업로드 즉시 실행, 결과 피드백

**경로 B — 갤러리 다중 선택**
1. 갤러리 버튼 탭
2. 갤러리에서 여러 장 선택 (다중 선택 UI)
3. 선택된 이미지 전체에 정제 검사 일괄 실행
4. 검사 결과 요약 표시 (통과 N장 / 제외 N장 + 사유)
5. 사용자 확인 후 통과 이미지 일괄 업로드

### 제약 사항
- 1회 선택 최대 업로드 수: 추후 결정 (초안 20장)
- 지원 포맷: JPEG, PNG, HEIC

---

## 기능 2. 내 업로드 목록 조회

### 목표
본인이 업로드한 사진을 시간 역순으로 확인한다.

### 동작 방식
- 그리드(3열) 형태로 썸네일 표시
- 무한 스크롤 페이지네이션
- 이미지 탭 시 원본 이미지 + 촬영 일시, 위치(GPS 포함 시) 상세 표시

---

## 기능 3. 이미지 정제

업로드 전 두 가지 검사를 수행한다. 검사는 **클라이언트 사이드**에서 실행해 불필요한 업로드 트래픽을 차단한다.

---

### 3-1. 중복 이미지 방지

#### 목표
동일한 사진을 중복으로 업로드하는 것을 방지한다.

#### 판별 기준 (우선순위 순)

| 단계 | 방법 | 설명 |
|------|------|------|
| 1차 | EXIF 메타데이터 비교 | 촬영 일시(`DateTimeOriginal`) + 기기 모델 + GPS 좌표(소수점 4자리 반올림) 조합이 동일하면 중복으로 판정 |
| 2차 | 파일 해시(SHA-256) 비교 | EXIF가 없는 이미지(스크린샷 등)에 대한 보조 수단 |

#### 처리 흐름
1. 업로드 시도 시 로컬 DB(SQLite)에 저장된 업로드 이력과 비교
2. 중복 의심 시 사용자에게 "이미 업로드한 사진과 유사합니다" 경고 표시
3. 사용자가 강제 업로드 선택 가능 (오탐 대응)

#### 고려사항
- EXIF가 제거된 이미지(SNS 다운로드 등)는 해시 비교만 적용
- GPS 비교는 EXIF에 위치 정보가 있을 때만 사용

---

### 3-2. 흔들린 사진 판별

#### 목표
블러(motion blur, defocus blur)가 심한 사진을 걸러낸다.

#### 판별 방법 — Laplacian 분산

이미지를 그레이스케일로 변환 후 Laplacian 필터를 적용해 분산값을 계산한다.  
분산값이 낮을수록 엣지(경계선)가 불분명 → 흔들린 사진으로 판정한다.

```
분산값 < 임계값(threshold) → 흔들린 사진
```

**임계값 결정 전략**
- 초기값: 실험적으로 100 ~ 150 사이에서 시작
- 실제 플로깅 환경(야외, 다양한 조도)에서 수집한 샘플로 튜닝 필요
- 임계값은 서버 또는 앱 설정에서 조정 가능하도록 외부화

**플랫폼별 구현**
- Flutter 순수 Dart로 구현하기에 연산 비용이 크므로 네이티브 코드 또는 isolate 사용 검토
- 패키지 후보: `image` (Dart), `opencv_dart` (OpenCV 바인딩)

#### 처리 흐름
1. 선택된 이미지를 썸네일 크기(예: 300px)로 리사이즈 후 분석 (성능 최적화)
2. 임계값 미만이면 해당 이미지를 "흔들린 사진" 목록으로 분류
3. 업로드 전 결과 요약 화면에서 제외 사진과 사유 표시
4. 사용자가 흔들린 사진도 포함해 업로드할 수 있는 옵션 제공 (오탐 대응)

#### 한계 및 보완
- Laplacian은 **노이즈가 많은 이미지**를 선명하다고 오판할 수 있음 → 노이즈 제거(Gaussian blur) 선처리 고려
- 야간 사진은 임계값을 낮춰 적용하는 것을 검토 (EXIF ISO 값 참고)

---

## 사용자 흐름 요약

```
[메인 화면]
    ├── [카메라 버튼] → 촬영 → 정제 검사 → 즉시 업로드
    └── [갤러리 버튼] → 다중 선택 → 정제 검사 → 요약 확인 → 일괄 업로드

[내 기록 탭]
    └── 업로드한 사진 그리드 조회 → 상세 보기
```

## 미결 사항

| 항목 | 내용 |
|------|------|
| 흔들림 임계값 | 샘플 데이터 수집 후 결정 |
| 1회 최대 업로드 수 | 서버 용량 정책에 따라 결정 |
| 중복 판정 강도 | GPS 오차 허용 범위(몇 미터?) 결정 필요 |
| 오프라인 지원 여부 | 촬영 후 네트워크 연결 시 자동 업로드 고려 |
| 블러 판별 연산 위치 | 클라이언트 단독 vs 서버 보조 판별 |
