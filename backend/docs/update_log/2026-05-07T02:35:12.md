### 1단계: Auth ✅
- 이메일/구글/카카오 로그인
- JWT (access + refresh) 토큰 관리
- 회원가입, 로그아웃, 비밀번호 찾기, 토큰 갱신

### 2단계: 트래킹 (GPS, 경로, 거리, 쓰레기 수거) ✅

#### 데이터 모델
- **TrackingSession**: 플로깅 활동 1회
  - id, user_id, status (active/paused/completed/expired)
  - started_at, ended_at, duration_seconds
  - paused_at, pause_duration_seconds (일시정지 관리)
  - distance_meters (Haversine으로 계산)
  - start/end 좌표 (lat/lng)
  - place_id, description (사용자 편집 가능)
  - trash_items (JSON: 카테고리별 수거량)
  - created_at, updated_at

- **TrackingPoint**: 실시간 GPS 포인트 (5초마다 저장)
  - session_id (FK), lat, lng, recorded_at

- **TrashPoint**: 실시간 쓰레기 마킹 (지도 마커용)
  - session_id (FK), lat, lng, category, note, recorded_at

- **Place**: 네이버 검색 결과 캐시
  - naver_place_id (unique), name, category, address, road_address, lat, lng

#### API 엔드포인트 (14개)

**기본 세션 관리 (8개)**
```
POST   /api/tracking/sessions                   세션 시작
GET    /api/tracking/sessions/active            진행 중 세션 확인 (이어하기용)
POST   /api/tracking/sessions/{id}/points       GPS 포인트 추가
POST   /api/tracking/sessions/{id}/end          종료 (쓰레기/장소 입력)
GET    /api/tracking/sessions                   목록 (페이지네이션, status 필터)
GET    /api/tracking/sessions/{id}              상세 (포인트 포함)
PATCH  /api/tracking/sessions/{id}              위치/설명 편집 (완료 후만)
DELETE /api/tracking/sessions/{id}              삭제
```

**일시정지/재개 (2개)**
```
POST   /api/tracking/sessions/{id}/pause        active → paused
POST   /api/tracking/sessions/{id}/resume       paused → active (pause_duration 누적)
```

**실시간 쓰레기 마킹 (2개)**
```
POST   /api/tracking/sessions/{id}/trash-points 마커 추가
GET    /api/tracking/sessions/{id}/trash-points 마커 목록 (지도 표시용)
```

**사용자 통계 (1개)**
```
GET    /api/users/me/stats                      누적 통계
```

**장소 검색 (1개)**
```
GET    /api/places/search?query=                네이버 지역 검색 (프록시)
```

#### 핵심 기능

**거리 계산**
- Haversine 공식: WGS84 기반 두 좌표 간 거리
- 포인트 추가 시 증분 합산
- 종료 시 전체 재계산으로 보정

**일시정지/재개**
- pause: active → paused, paused_at 기록
- resume: paused_duration_seconds에 (now - paused_at) 누적, 상태 복귀
- end: duration = 전체경과 - pause_duration (정확한 활동 시간만 계산)
- GPS 포인트는 active일 때만 추가 가능 (paused 상태에서 거부)
- 쓰레기 마킹은 active/paused 둘 다 가능

**만료 관리 (1시간)**
- active/paused 세션이 시작 후 1시간 초과 시 expired로 자동 전환
- paused에서 만료될 경우 현재 일시정지 구간도 누적

**쓰레기 수거 기록**
- trash_items: 종료 시 입력하는 카테고리별 수거량 (level 또는 count)
  - level: "little"(1-10) → 5, "moderate"(11-30) → 20, "a_lot"(31+) → 40
  - count: 직접 입력 (둘 중 하나만)
- trash_points: 활동 중 실시간으로 마킹하는 좌표 + 카테고리 (지도용)

**네이버 지역 검색**
- 엔드포인트: https://openapi.naver.com/v1/search/local.json
- 헤더: X-Naver-Client-Id, X-Naver-Client-Secret
- 응답 좌표: WGS84 × 10^7 (정수형) → 자동 변환
- HTML 태그(`<b>`) 제거하고 반환
- 응답 제약: display 최댓값 5, start 최댓값 1 (페이지네이션 불가)

**사용자 통계**
- completed 세션만 집계
- total_distance_meters: 누적 거리
- total_duration_seconds: 누적 활동 시간 (pause_duration 제외)
- total_sessions: 완료된 세션 수
- total_trash_count: trash_items 기반 (level → 대표값, count → 합산)

#### 마이그레이션 전략
- SQLAlchemy `create_all()`: 새 테이블 생성
- SQLite `ALTER TABLE`: 기존 테이블에 새 컬럼 추가 (paused_at, pause_duration_seconds)
- lifespan에서 idempotent하게 실행 → 기존 사용자 데이터 보존

#### 테스트 결과 ✅
- 세션 생성 → 포인트 추가 (거리 76m) → 일시정지
  - pause 중 포인트 추가 거부 (409)
  - pause 중 쓰레기 마킹 추가 가능 (201)
- 2초 대기 → 재개 (pause_duration_seconds=2 누적)
- 포인트 추가 (거리 131m → 총 207m)
- 종료 (duration=0, pause_duration=2)
- 쓰레기 마킹 1개 조회 가능
- 통계: total_distance=207, total_sessions=1, total_trash_count=27 (moderate=20+count=7)
